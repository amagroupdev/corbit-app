# توثيق API لمنصة أوربت SMS V3

## معلومات عامة

| البند | القيمة |
|-------|--------|
| الرابط الأساسي (Production) | `https://app.mobile.net.sa/api/v3` |
| الرابط الأساسي (Staging) | `https://staging.mobile.net.sa/api/v3` |
| نوع المصادقة | Bearer Token |
| صيغة البيانات | JSON |
| مهلة الاتصال | 30 ثانية |
| مهلة الإرسال | 30 ثانية |
| مهلة الاستقبال | 30 ثانية |

## الهيدرز المطلوبة

```
Accept: application/json
Content-Type: application/json
Authorization: Bearer {access_token}
Accept-Language: ar   (أو en)
```

## الترقيم (Pagination)

جميع قوائم البيانات تدعم الترقيم:

| المعامل | الوصف | القيمة الافتراضية |
|---------|-------|-------------------|
| `page` | رقم الصفحة | 1 |
| `per_page` | عدد العناصر في الصفحة | 15 |
| `search` | نص البحث | - |
| `sort_by` | الحقل المراد الترتيب به | - |
| `sort_order` | اتجاه الترتيب (asc/desc) | - |
| `date_from` | من تاريخ | - |
| `date_to` | إلى تاريخ | - |
| `status` | فلترة حسب الحالة | - |
| `sender_id` | فلترة حسب المرسل | - |
| `group_id` | فلترة حسب المجموعة | - |
| `type` | فلترة حسب النوع | - |
| `locale` | اللغة | - |

الحد الأقصى لعدد العناصر في الصفحة: **100**

## 1. التسجيل والمصادقة (Auth)

### 1.1 تسجيل الدخول
```
POST /auth/login
```
**المدخلات:**
| الحقل | النوع | مطلوب | الوصف |
|-------|-------|-------|-------|
| `username` | string | نعم | اسم المستخدم أو رقم الجوال |
| `password` | string | نعم | كلمة المرور |

**المخرجات:**
```json
{
  "access_token": "eyJ...",
  "user": { ... }
}
```
**ملاحظة:** إذا كان الحساب مفعل فيه التحقق بخطوتين (2FA)، سيعيد رمز مؤقت بدل التوكن ويطلب التحقق من الرمز.

---

### 1.2 تسجيل حساب جديد
```
POST /auth/register
```
**المدخلات:**
| الحقل | النوع | مطلوب | الوصف |
|-------|-------|-------|-------|
| `name` | string | نعم | الاسم الكامل |
| `email` | string | نعم | البريد الإلكتروني |
| `username` | string | نعم | اسم المستخدم (3-30 حرف) |
| `password` | string | نعم | كلمة المرور (8 أحرف على الأقل، حرف كبير، حرف صغير، رقم، رمز خاص) |
| `password_confirmation` | string | نعم | تأكيد كلمة المرور |
| `phone` | string | نعم | رقم الجوال |
| `gender` | string | لا | الجنس (male/female) |
| `region_id` | int | لا | معرف المنطقة |
| `city_id` | int | لا | معرف المدينة |
| `organization_type_id` | int | لا | نوع المنظمة (للحسابات المؤسسية) |
| `company_name` | string | لا | اسم الشركة/المدرسة/الجهة |

**المخرجات:** بيانات المستخدم + رسالة نجاح، ثم يُطلب التحقق من رقم الجوال عبر OTP.

---

### 1.3 التحقق بخطوتين (2FA)

| `phone` | string | نعم | رقم الجوال |

## 4. الرسائل (Messages)

### 4.1 إرسال رسالة
```
POST /messages/send
```
**المدخلات:**
| الحقل | النوع | مطلوب | الوصف |
|-------|-------|-------|-------|
| `sender_id` | int | نعم | معرف اسم المرسل |
| `body` | string | نعم | نص الرسالة |
| `numbers` | array | شرطي | أرقام المستقبلين (مطلوب إذا لم تُحدد مجموعات) |
| `group_ids` | array | شرطي | معرفات المجموعات (مطلوب إذا لم تُحدد أرقام) |
| `scheduled_at` | datetime | لا | تاريخ ووقت الجدولة (إذا لم يُحدد يُرسل فوراً) |
| `type` | string | لا | نوع الرسالة |

**المخرجات:**
```json
{
  "message": "تم إرسال الرسالة بنجاح",
  "data": {
    "numbers_count": 150,
    "sms_count": 300,
    "cost": 300
  }
}
```
